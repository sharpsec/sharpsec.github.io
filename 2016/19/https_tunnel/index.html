https_tunnel BarracudaDrive 折腾小记

发表于 2016-09-19   |   分类于 协议安全   |     |   阅读次数

开始搜集 https_tunnel 相关的工具，但是国内几乎很少，除了 metaspolit 中 reverse_https 反弹木马，你可以尝试一下，今天在 google 上看到了一款工具 MxTunnel，原谅我英文学的不好，折腾了一下午，开始实验;

环境准备

开始实验

  1. 先下载服务端，客户端可通过服务端程序生成（这是我实验的方式，还有独立的客户端 MxTunnel.exe,有命令行和 UI 的，你可以尝试),安装好的界面如下；

  2. 注册一个账号，点击 Memu,左边第一项就是 https_tunnel,就此服务器傻瓜式安装结束；

  1. 客户端通过浏览器访问（访问过程中可能转成 https 加密传输，这个非常牛叉），注意端口正确（我一开始 Web 端口9357），同样打开 Memu，选择 https_tunnel,点击高级选项，add_tunnel,如下配置，将服务器 3389 端口映射到本地 4444 端口，start_tunnel 会生成一个 jnlp 文件，这就是马上生成客户端一个xml；
  2. 坑来了，安装好java 环境运行 jnlp 文件会出现证书撤销和错误问题；

问题解决

  1. 打开控制面板中java选项
  2. 选择高级如下配置
  3. 启动之后会生成的一个客户端，如果碰到端口冲突，就换成不常用的随意端口；

执行成功

  1. 可以远程连接本机的 4444 端口，即可连接到服务器；

  2. 抓取流量包看看，的确是 https 协议包，结束了；
